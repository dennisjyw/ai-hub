# 解析審查

## 功能概述
此技能用於優化題目解析內容，透過「教育專家改寫 + 審查委員檢核」的雙層流程，確保解析內容原創性、書寫規範一致性與格式正確性。

## 核心功能
- **解析改寫**：改寫 explanation、explanation_a/b/c/d 欄位，確保查重率低於 50%。
- **書寫規範檢查**：確保中文與英文/數字間有半形空格，標點符號全形/半形正確。
- **下劃線一致性**：統一所有填空題的下劃線長度。

## 觸發範例
可直接對助手說：
- 「請針對這份 CSV 的解析欄位進行優化改寫。」
- 「幫我檢查題目的書寫規範，確保查重率低於 50%。」
- 「統一所有填空題的下劃線長度。」

## 輸入資料建議
- `questions.csv`（題目匯總表）
  - 必要欄位：`question_stem`、`option_a`/`option_b`/`option_c`/`option_d`
  - 解析欄位：`explanation`、`explanation_a`/`explanation_b`/`explanation_c`/`explanation_d`

## 標準輸出
- `optimized_questions.csv`：優化後的題目資料（欄位與原始題庫 CSV 或標準題庫欄位一致）
- `optimization_report.md`：優化統計報告
- `changes_log.csv`：修改紀錄

## 工作流程
1. 教育專家分析原始解析語義，改寫內容確保查重率低於 50%。
2. 審查委員檢核語義一致性、書寫規範與格式正確性。
3. 輸出優化後的 CSV 與報告。

## 專業團隊

| 角色 | 職責 |
|------|------|
| 教育專家 | 分析原意、改寫解析、保持教學價值 |
| 審查委員 | 檢核語義一致性、查重率、書寫規範 |

## 注意事項
- 改寫必須實質性重寫，不能僅替換少數詞彙。
- 改寫過程中絕對不能改變解析的核心意思。
- 學術專有名詞可保留原文，不強制改寫。
- 若此流程接在 `scripts/pipeline/` 後執行，輸出 CSV 欄位順序應維持與來源 CSV 或標準題庫欄位完全一致。
