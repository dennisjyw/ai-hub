# 知識樹建立與重整

## 何時使用

- 從課綱、歷屆試題、教材目錄或既有題庫建立知識樹。
- 將既有 `level_1`、`level_2`、`level_3` 重整為新的 schema。
- 需要對齊其他科目知識樹的欄位、分類粒度或命名風格。

## 操作流程

1. 先盤點同工作區的既有知識樹或參考檔。
   - 讀欄位名稱、層級數、空白欄位是否代表沿用上一層。
   - 抽樣 `L3` 或 `level_3` 的命名長度與詞性。
2. 讀 source of truth。
   - 若使用者提供課綱、規範或教材，先抽取可檢索文字。
   - 不要直接照抄課綱條文；要整理成題庫標註可用的穩定概念節點。
3. 用歷屆試題或既有題庫校準粒度。
   - `L1` / `level_1` 放大類或科目領域。
   - `L2` / `level_2` 放單元群、章節群或課綱主題。
   - `L3` / `level_3` 放最小穩定概念，應短、可重用、可標註。
4. 若參考檔採 `Subject,L1,L2,L3`：
   - 沿用同欄位名稱。
   - 同一 `Subject`、`L1`、`L2` 下方列可用空白表示沿用上一層。
   - `Subject` 的取值必須依使用者指定或既有科目分類，不自行擴充。
5. 命名要符合參考檔粒度。
   - `L3` 優先用短名詞，例如「光合作用」「歐姆定律」「月相與食」。
   - 避免長句、條文式描述，像「日光二氧化碳水分與光合作用速率」應縮成「光合速率」。
   - 已可由 `L1` / `L2` 推得出的字，不要重複塞進 `L3`。
6. 回讀驗證。
   - 檢查欄位數。
   - 檢查必填欄位空值。
   - 若有空白沿用上一層，做 forward-fill 後檢查完整路徑。
   - 檢查完整路徑重複。
   - 檢查 `Subject` 或 `level_1` 是否只含允許值。
   - 抽樣檢查 `L3` 平均長度與最長長度是否接近參考檔。

## 國中會考自然科案例

這次在 `parsing-nature` 的有效流程：

1. 先讀 `references/` 內國文、英語、數學知識樹。
2. 確認參考格式為 `Subject,L1,L2,L3`，且多數列用空白欄位沿用上一層。
3. 讀自然領綱 PDF 與 102-114 年自然科試題本，確認分類不是直接抄課綱，而是為題庫標註重組。
4. 依使用者指定將 `Subject` 固定為 `生物`、`物理`、`化學`、`地科`。
5. 將原本較長的 `L3` 條文式描述改成短名詞。
6. 驗證結果包含：
   - `Subject,L1,L2,L3` 欄位一致。
   - `Subject` 只含四科或空白。
   - `L3` 無空值。
   - forward-fill 後完整路徑無重複。
   - CSV 每列欄位數一致。
