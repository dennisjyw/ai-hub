# 批次解析撰寫流程

## 功能概述
用於批次撰寫題目的 explanation、explanation_a/b/c/d 欄位，採用「分段執行、定期審查」模式，確保品質穩定且易於追蹤進度。

## 工作流程

### 階段架構

```
┌─────────────────────────────────────────────────────────────────────┐
│                         批次解析撰寫流程                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Phase 1: 第 1-10 題  ──→  Phase 2: 第 11-20 題  ──→  Review 1    │
│       ↓                           ↓                      ↓         │
│  撰寫 explanation          撰寫 explanation          檢查品質       │
│  撰寫選項解析               撰寫選項解析               一致性確認    │
│                                                                     │
│  Phase 3: 第 21-30 題  ──→  Phase 4: 第 31-40 題  ──→  Review 2    │
│       ↓                           ↓                      ↓         │
│  撰寫 explanation          撰寫 explanation          檢查品質       │
│  撰寫選項解析               撰寫選項解析               一致性確認    │
│                                                                     │
│  ...持續直到全部完成...                                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 詳細步驟

1. **確認範圍**
   - 總題數：確認 CSV 總共有多少題
   - 目標欄位：explanation、explanation_a/b/c/d
   - 已存在解析：是否需要重新撰寫或補充

2. **分段執行（每 10 題為一個階段）**
   - 每次處理 10 題的完整解析
   - 依序撰寫：
     - `explanation`：整體解析，說明核心概念與答案判斷依據
     - `explanation_a`：選項 A 的正誤解析
     - `explanation_b`：選項 B 的正誤解析
     - `explanation_c`：選項 C 的正誤解析
     - `explanation_d`：選項 D 的正誤解析

3. **定期審查（每 20 題 Review 一次）**
   - 完成第 20、40、60... 題後暫停
   - 進行品質檢查：
     - 解析是否清楚說明核心概念
     - 各選項解析是否明確指出正誤原因
     - 用語是否一致（同一章節的概念用詞需統一）
     - 格式是否符合規範（中英間距、標點符號等）
   - 發現問題時，先修正再繼續下一批

4. **輸出與存檔**
   - 每個階段完成後更新 CSV 檔案
   - 記錄進度：已完成題數 / 總題數
   - 儲存備份：`{filename}_backup_{phase}.csv`

## 撰寫原則

### 整體解析 (explanation)
1. 先說明題目考查的核心概念
2. 再說明答案判斷的依據
3. 適當引用題目中的條件或情境
4. 字數建議：80-150 字

### 選項解析 (explanation_a/b/c/d)
1. **正確選項**：說明為何符合題意，引用相關概念或定義
2. **錯誤選項**：說明為何不符合，指出常見錯誤或迷思
3. 每個選項解析字數建議：30-60 字
4. 避免只寫「正確」或「錯誤」，需有實質說明

### 一致性要求
- 同一章節的專有名詞需統一（如「光合作用」vs「光合成」）
- 數字、符號、單位的表示方式需一致
- 語氣風格保持一致（正式、教學口吻）

```markdown
### 進度

- 累計完成：{total_done} / {total} 題
- 完成率：{percentage}%

### 本次處理題號
{question_numbers}

### 輸出檔案
- `{filename}.csv`：已更新解析欄位
- `{filename}_backup_phase{phase}.csv`：階段備份

### 撰寫統計
| 欄位 | 本次新增 | 累計新增 |
|------|----------|----------|
| explanation | {count} | {total} |
| explanation_a | {count} | {total} |
| explanation_b | {count} | {total} |

```

### Review 報告
```markdown
## 第 {start}-{end} 題 Review 報告

### 檢查項目
- [x] 解析完整性：每題都有 explanation 與選項解析
- [x] 概念說明清晰度：核心概念是否清楚闡述
- [x] 選項解析準確性：正誤原因說明是否正確
- [x] 用語一致性：專有名詞是否統一
- [x] 格式規範：中英間距、標點符號

### 發現問題
| 題號 | 欄位 | 問題 | 修正方式 |
|------|------|------|----------|
| {id} | {field} | {issue} | {fix} |

### 修正摘要
- 問題數：{issue_count}
- 已修正：{fixed_count}
- 狀態：通過 / 需再次檢查
```

## 注意事項

1. **優先延續既有風格**：若題庫已有部分解析，先閱讀並模仿其風格
2. **遇到困難題目**：若遇到無法判斷的題目，標記並詢問使用者
3. **保存進度**：定期存檔，避免意外中斷需重新開始
4. **使用者中斷**：使用者可以隨時說「暫停」，下次可從中斷點繼續
