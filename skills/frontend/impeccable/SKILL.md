# Impeccable - 前端設計總控

智能路由引擎，理解自然語言設計需求，自動匹配並調用子技能。

## 何時使用

- 需要前端設計的全流程支援
- 自然語言描述設計需求（如「讓設計更大膽」、「簡化介面」）
- 需要調用特定設計子技能（adapt、animate、audit、bolder 等）

## 核心理念

- **無需記憶命令**：用自然語言描述需求即可
- **自動匹配技能**：根據語義選擇最合適的子技能
- **可疊加調用**：支援連續調用多個技能完成複雜任務

## 子技能矩陣

| 子技能 | 用途 | 觸發關鍵詞 |
|--------|------|-----------|
| adapt | 響應式適配 | 移動端、響應式、跨設備 |
| animate | 動畫效果 | 動畫、過渡、微交互 |
| audit | 技術審計 | 檢查、審計、可訪問性 |
| bolder | 視覺衝擊 | 太平淡、更大膽 |
| clarify | 文案優化 | 文案、提示、清晰 |
| colorize | 色彩策略 | 顏色、配色、太灰 |
| critique | 設計評審 | 評審、反饋、評估 |
| delight | 愉悅體驗 | 驚喜、有趣、個性化 |
| distill | 簡化設計 | 簡化、太複雜、精簡 |
| harden | 生產加固 | 上線、錯誤處理、邊界情況 |
| layout | 佈局優化 | 佈局、間距、層次 |
| optimize | 性能優化 | 性能、速度、卡頓 |
| overdrive | 技術突破 | 震撼、炫酷、前沿 |
| polish | 最終打磨 | 打磨、完善、細節 |
| quieter | 降低強度 | 太吵、太花、柔和 |
| shape | UX規劃 | 規劃、簡報、方向 |
| typeset | 排版優化 | 字體、排版、可讀性 |

## 使用方式

### 直接描述需求
用自然語言描述設計需求，系統自動匹配技能：
- 「這個頁面在手機上顯示有問題」→ 調用 adapt
- 「設計看起來太平淡了」→ 調用 bolder
- 「幫我檢查一下可訪問性」→ 調用 audit

### 顯式指定技能
- 「用 adapt 技能處理移動端適配」
- 「調用 animate 添加動畫效果」

### 鏈式調用
「先優化佈局，然後添加動畫，最後打磨細節」→ 依序調用 layout、animate、polish

## 設計原則

- **拒絕 AI 審美**：避免千篇一律的 AI 生成風格
- **大膽且有意**：設計要有明確的意圖和個性
- **細節至上**：每個像素都經過精心考量
- **用戶中心**：設計服務於用戶目標
