# Tailwind Polish

## 何時使用
- Tailwind CSS 細修與視覺打磨。
- 已有畫面，但需要更穩、更精緻、更一致。

## 聚焦範圍
- 間距、字級、層級、邊框、陰影、配色與狀態。
- 避免常見 AI 味：卡片套卡片、灰字壓彩底、視覺重量失衡。

## 執行流程
1. 先指出目前畫面的主要視覺問題。
2. 優先改高影響的小地方：排版、節奏、對比、互動狀態。
3. 只在必要時延伸到 layout 或元件結構。

## 產出要求
- 說明每個 Tailwind 調整要解決什麼視覺問題。
- 優先用現有 tokens、utility 與設計系統，不亂加自訂樣式。
