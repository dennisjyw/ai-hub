# JSON Canvas

## 何時使用
- `.canvas` 檔新增、修改、排版。
- 節點、邊、群組、連線與視覺流程整理。

## 載入策略
- 先用主技能流程處理。
- 需要範例時再看 [references/EXAMPLES.md](references/EXAMPLES.md)。

## 執行流程
1. 先確認是新建還是修改既有 canvas。
2. 先定義節點類型與布局，再補連線與群組。
3. 修改既有檔案時，避免打亂未涉及區塊的座標與結構。
4. 交付前確認 JSON 合法、節點 id 唯一、邊的來源與目標存在。
