# Literature & IMRaD Structure Analyzer
# 文獻閱讀與 IMRaD 結構分析技能

## 簡介

這是一個專為碩博士生與研究人員設計的技能,幫助系統化地閱讀和分析學術論文。

### 核心功能

1. **IMRaD 結構拆解**: 自動識別並分析論文的 Introduction, Method, Results, Discussion 四大部分
2. **研究方法提取**: 清楚整理研究設計、樣本、變項、測量工具、統計分析
3. **關鍵資訊彙整**: 提取研究限制、未來方向、理論貢獻
4. **方法論學習**: 提供學習重點,幫助理解研究設計邏輯

### 為什麼需要這個技能?

**IMRaD 不只是格式,更是學術思考的邏輯。**

要內化這種思維方式,需要長期反覆練習「怎麼看、怎麼拆、怎麼整理」。這個技能提供系統化的分析框架,幫助你:

- 快速掌握論文的核心架構
- 學習如何設計研究(從別人的方法中學習)
- 找到研究限制(未來研究的起點)
- 建立批判性閱讀的習慣

## 使用方式

### 基本使用

1. 上傳 PDF 格式的學術論文
2. 說「請幫我分析這篇論文的 IMRaD 結構」
3. 獲得完整的結構化分析報告

### 進階使用

#### 1. 聚焦特定部分

```
「我特別想了解這篇論文的研究方法」
「幫我整理這篇論文的研究限制」
「這篇的統計分析用了什麼方法?」
```

#### 2. 指定輸出格式

```
「請產出 Word 檔案讓我可以編輯」
「用中英雙語分析」
```

#### 3. 批次分析多篇論文

```
「我上傳了三篇相關論文,請比較它們的研究方法」
```

## 輸出內容

分析報告包含以下部分:

### 1. Introduction 分析
- 研究問題/缺口
- 文獻回顧重點
- 研究目的與假設

### 2. Method 分析
- 研究設計類型
- 樣本特徵(N、抽樣方法)
- 變項定義(IV、DV、控制變項)
- 測量工具
- 資料分析方法

### 3. Results 分析
- 主要發現
- 統計結果(具體數值)
- 圖表重點

### 4. Discussion 分析
- 結果詮釋
- 與過往研究比較
- **研究限制** ⚠️
- 理論/實務意涵
- 未來研究方向

### 5. 整合摘要
一頁速覽表格,包含核心研究問題、方法、發現、限制

### 6. 方法論學習重點
針對研究設計、變項操作化、資料分析的學習要點

## 輸出格式

- **預設**: Markdown 格式 (.md)
- **可選**: Word 文檔 (.docx) - 需明確要求
- **語言**: 繁體中文(預設)或英文
- **專有名詞**: 中英對照

## 適用論文類型

### ✅ 適合

- 量化研究論文
- 質性研究論文
- 混合方法研究
- 實證研究論文

### ⚠️ 有限支援

- 理論型文章(會盡量對應到 IMRaD 邏輯)
- 文獻回顧論文(結構可能不標準)
- 系統性回顧(會說明實際結構)

### ❌ 不適用

- 非學術文章
- 書籍章節(結構差異過大)
- 僅摘要的文件

## 技能限制

本技能**不包含**:

- ❌ 文獻搜尋(如何找論文)
- ❌ 文獻管理(整理引用格式)
- ❌ 論文撰寫(寫自己的論文)
- ❌ 統計驗證(重新計算統計結果)
- ❌ 全文翻譯

## 範例互動

### 範例 1: 標準分析

**使用者**: [上傳 PDF] 幫我分析這篇論文的結構

**輸出**: 
- 完整的 IMRaD 四部分分析
- 整合摘要表格

### 範例 3: 產出 Word 檔案

**使用者**: 請分析這篇論文並產出 Word 檔讓我編輯

- 完整 IMRaD 分析

- 可直接編輯

## 學習建議

使用這個技能時,建議:

1. **對照原文閱讀**: 分析報告是輔助工具,不是替代品
2. **批判性思考**: 思考「如果是我,會怎麼設計研究?」
3. **累積方法庫**: 整理不同研究方法的優缺點
4. **關注限制**: 限制就是未來研究的機會
5. **建立模板**: 多讀幾篇後,建立自己的閱讀檢查清單
