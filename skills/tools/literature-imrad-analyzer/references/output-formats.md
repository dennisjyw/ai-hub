# IMRaD 結構分析輸出格式

## Introduction 分析

```markdown
## Introduction 分析

### 研究問題
- [簡述核心研究問題]

### 文獻缺口
- [指出現有研究的不足之處]

### 研究目的
- [列出本研究的具體目標]
```

## Method 分析

```markdown
## Method 分析

### 研究設計
- [研究類型與設計]

### 樣本
- 樣本數: N = [數字]
- 抽樣方法: [說明]
- 特徵: [描述]

### 變項
**自變項:**
- [變項名稱]: [操作型定義]

**依變項:**
- [變項名稱]: [測量方式]

### 測量工具
- [工具名稱]: [簡述內容與信效度]

### 料分析
- [統計方法清單]
```

## Results 分析

```markdown
## Results 分析

### 主要發現
1. [發現 1 + 統計數據]
2. [發現 2 + 統計數據]

### 圖表解讀
- **Figure/Table [編號]**: [圖表主要呈現的資訊]

### 額外發現
- [次要但值得注意的結果]
```

## Discussion 分析

```markdown
## Discussion 分析

### 主要論點
- [作者對結果的核心詮釋]

### 與文獻對話
- **支持的研究**: [引用]
- **矛盾的研究**: [引用] - [作者如何解釋差異]

### 研究限制 ⚠️
1. [限制 1]: [可能的影響]
2. [限制 2]: [可能的影響]

### 貢獻與啟示
- **理論貢獻**: [說明]
- **實務意涵**: [說明]

### 未來研究建議
- [建議方向 1]
- [建議方向 2]
```

## 整合摘要

```markdown
## 📊 整合摘要:一頁速覽

| 項目 | 內容 |
|------|------|
| **核心研究問題** | [一句話描述] |
| **研究方法** | [研究設計類型] |
| **樣本** | N=[數字], [特徵] |
| **主要變項** | IV: [列出] / DV: [列出] |
| **關鍵發現** | [2-3 個最重要的發現] |
| **主要限制** | [最重要的 1-2 個限制] |
| **可借鑑之處** | [這篇論文可以給你的啟發] |
```

## 方法論學習重點

```markdown
## 🎯 方法論學習重點

### 這篇論文的方法論可以學什麼?

1. **研究設計**:
   - [這篇使用的設計有什麼優點?]
   - [什麼情況適合使用這種設計?]

2. **變項操作化**:
   - [作者如何將抽象概念轉化為可測量的變項?]
   - [測量工具的選擇邏輯?]

3. **資料分析策略**:
   - [為什麼選擇這些統計方法?]
   - [分析步驟的邏輯順序?]

4. **可以改進的地方**(批判性思考):
   - [如果是你,你會怎麼調整研究設計?]
```
