# Design Library

## 何時使用
- 使用者明確提到要像某個網站、品牌或產品。
- 需要找 UI 參考、視覺方向、版型語言、互動風格。
- 想從現有 reference library 挑設計母板。

## 工作原則
- 先選 1 到 3 個最接近的參考，不要一次混太多風格。
- 先提煉可重用原則，再落到字體、顏色、卡片、hero、CTA。
- 說清楚「借的是語言」，不是逐像素抄襲。

## 載入策略
- 先在 `references/` 中選最相關品牌資料夾。
- 每個品牌優先讀 `README.md`；只有需要深挖細節時才讀 `DESIGN.md`。
- `preview.html` 與 `preview-dark.html` 只在需要視覺確認時打開。

## 執行流程
1. 依產業、氣質、明暗模式與產品型態挑參考。
2. 提煉設計原則：版面密度、字體語氣、配色、元件輪廓、互動節奏。
3. 轉成專案可落地的 UI 指引或實作建議。

## 產出要求
- 至少交付：參考名單、為何選它、要借哪些元素、要避免哪些元素。
- 若使用者要實作，補充成具體元件與頁面規則。
